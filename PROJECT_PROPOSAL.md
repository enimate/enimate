# Enimate - 2D 动画制作工具

## 项目申报书

---

## 一、项目目标

**本次 MVP 目标**：

> **能画、能动、能导出**

- **能画**：提供基础矢量绘图工具（矩形、椭圆、线条、贝塞尔曲线）
- **能动**：时间轴、图层、关键帧、洋葱皮
- **能导出**：PNG 序列帧

**远期规划（不在本次范围内）**：

长远来看，本项目希望成为"AI 可以操作的动画工具"。如果动画数据模型足够清晰，AI Agent 就能通过 API 自动创建时间轴、插入关键帧、生成动画。这是未来的方向，本次 MVP 不涉及，但数据模型设计时会预留扩展空间。

## 二、交付物说明

### 2.1 核心功能

| 模块 | 功能 | 优先级 |
|------|------|--------|
| **绘图工具** | 选择、矩形、椭圆、线条、钢笔（贝塞尔曲线）、手型（平移） | P0 |
| **图层系统** | 新建/删除/重命名、显示/隐藏/锁定、顺序调整 | P0 |
| **时间轴** | 关键帧插入/删除、空白关键帧、播放控制、帧率设置 | P0 |
| **洋葱皮效果** | 显示前后 N 帧（半透明叠加） | P0 |
| **导出** | PNG 序列帧导出（逐帧渲染保存） | P0 |
| **项目文件** | 新建/打开/保存项目（.enim 格式） | P1 |
| **图形变换** | 缩放、旋转（选择工具扩展） | P1 |

### 2.2 功能边界（MVP 明确不做）

| 功能 | 原因 |
|------|------|
| 骨骼动画 | 复杂度过高，MVP 聚焦逐帧 |
| 脚本系统 | 使用Moonbit作为脚本，类似Animate使用ActionScript、Blender使用Python |
| 补间动画 | 需要复杂的插值算法 |
| 铅笔工具（自由绘制） | 需要位图系统而非矢量，涉及笔刷设计、压感支持、笔触平滑等，复杂度高 |
| 音频同步 | 需要音频引擎集成 |
| 渐变填充 | UI 复杂，可延后 |
| 滤镜效果 | 性能优化复杂 |
| GIF/MP4直接导出 | 导出的PNG序列可依赖外部库转为GIF/MP4 |

### 2.3 预期使用方式

```
用户工作流：

1. 新建项目 → 设置画布尺寸、帧率
2. 绘制内容 → 选择工具绘制图形
3. 管理图层 → 添加/调整图层顺序
4. 编辑时间轴 → 插入关键帧、调整播放
5. 预览动画 → 播放/暂停/洋葱皮
6. 导出 → PNG 序列帧 → 后期合成视频/GIF
```

## 三、技术路线说明

### 3.1 整体系统架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        应用层 (Application)                       │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────────────────┐│
│  │ 工具系统  │ │ 时间轴    │ │ 图层管理 │ │    导出/持久化        ││
│  └──────────┘ └──────────┘ └──────────┘ └──────────────────────┘│
├─────────────────────────────────────────────────────────────────┤
│                        文档模型 (Document Model)                  │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────────────────┐│
│  │ Document │ │  Layer   │ │  Frame   │ │       Shape          ││
│  └──────────┘ └──────────┘ └──────────┘ └──────────────────────┘│
├─────────────────────────────────────────────────────────────────┤
│                        渲染层 (Rendering Layer)                   │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │  SkiaCanvas / SkiaPath (直接封装)                        │   │
│  └──────────────────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────────────────┤
│                        平台层 (Platform Layer)                    │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │  GLFW (窗口) │ OpenGL (显示) │ 系统库 (FFI)               │   │
│  └──────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 核心模块划分

| 模块 | 职责 | 关键类型 |
|------|------|----------|
| **文档模型** | 数据结构定义与操作 | `Document`, `Layer`, `Frame`, `Shape` |
| **绘图工具** | 用户交互与绘制逻辑 | `Tool` ADT, `SelectTool`, `RectangleTool` |
| **时间轴** | 动画时间控制 | `Timeline`, `Keyframe`, `PlaybackController` |
| **渲染层** | 图形绘制与显示 | `SkiaCanvas`, `SkiaPath`, `Paint` |
| **UI 层** | 界面组件 | Dear ImGui 集成 |
| **导出** | 文件输出 | PNG 序列帧导出器 |
| **持久化** | 项目保存/加载 | `.enim` 文件格式 |

### 3.3 大模型与智能体工具的作用

**开发过程中 AI 辅助的应用**：

| 阶段 | AI 辅助方式 | 具体应用 |
|------|-------------|----------|
| **需求分析** | 技术调研、竞品分析 | 分析 Adobe Animate 功能、对比现有开源工具 |
| **架构设计** | 设计评审、方案优化 | 文档模型设计、渲染层架构讨论 |
| **代码生成** | 代码编写、重构 | FFI 绑定代码、数据结构定义 |
| **问题排查** | 错误诊断、调试建议 | 编译错误、运行时问题分析 |
| **文档编写** | 文档生成、润色 | 用户手册、API 文档 |

**AI 辅助的关键成果**：

2. **工程文件格式设计**：借鉴 .pen 格式，设计 `.enim` 文件格式
3. **渲染抽象层设计**：预留后端扩展能力
4. **MVP 简化策略**：识别过度设计，简化为 Skia CPU 方案

### 3.4 关键技术选型

| 技术选型 | 选择 | 理由 |
|----------|------|------|
| **编程语言** | MoonBit | 函数式语言，支持 Native 编译，类型安全，FFI 支持良好 |
| **渲染后端** | Skia (CPU) | Google 维护的高质量 2D 图形库，跨平台，PNG 内置编码 |
| **窗口管理** | GLFW | 轻量级跨平台窗口库，OpenGL 上下文管理 |
| **UI 框架** | Dear ImGui | 即时模式 GUI，适合工具类应用，集成简单 |
| **文件格式** | JSON (.enim) | 人类可读，便于调试和版本控制 |

## 四、风险分析与应对方案

### 4.1 技术风险

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| **Skia 编译复杂** | 中 | 高 | 提供预编译库，详细构建文档 |
| **MoonBit FFI 稳定性** | 中 | 中 | 封装边界清晰，便于切换实现 |
| **性能不足** | 低 | 中 | MVP 使用 CPU 光栅化，预留 GPU 升级路径 |
| **跨平台兼容** | 中 | 中 | 优先 Linux，逐步支持 macOS/Windows |
| **UI 交互体验** | 中 | 低 | 参考 Animate/Pencil2D 交互设计 |

### 4.2 工程风险

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| **功能范围膨胀** | 高 | 高 | 严格 MVP 边界，P2 功能延后 |
| **开发周期延误** | 中 | 中 | 分阶段交付，核心功能优先 |
| **依赖库版本冲突** | 低 | 低 | 使用静态链接，锁定版本 |

### 4.3 替代方案

| 风险场景 | 替代方案 |
|----------|----------|
| Skia 编译失败 | 使用 Skia 预发布版本或系统包管理器安装 |
| MoonBit FFI 问题 | 增加中间 C Bridge 层，减少直接 FFI 调用 |
| 性能不达标 | 提前引入渲染抽象层，启用 GPU 后端 |
| Dear ImGui 集成困难 | 考虑使用原生 GUI（GTK/Qt） |

### 5.2 技术现状理解

**Skia 渲染引擎**：
- Google 开源 2D 图形库，被 Chrome、Flutter、Android 使用
- 支持 CPU 光栅化和 GPU 加速（Vulkan/Metal/OpenGL）
- 内置图像编码（PNG/JPEG），无需额外依赖

**Dear ImGui**：
- 业界广泛使用的即时模式 GUI 库
- 适合工具类应用开发
- 与图形渲染集成简单

## 八、总结

Enimate 旨在开发一款**轻量级、开源免费**的 2D 逐帧动画制作工具，填补 Adobe Animate 停止支持后的市场空白。项目采用 **MoonBit + Skia** 技术栈，MVP 阶段聚焦核心功能（绘图、图层、时间轴、导出），通过 AI 辅助开发在 4 周内完成交付。

更重要的是，本项目建立了**可编程的动画数据模型**，为未来的**AI Agent 自动生成动画**奠定基础，具有长远的技术价值。
